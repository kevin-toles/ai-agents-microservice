### Why This Matters for Services

**Semantic Search** and **AI Agents** must understand that:

1. **Relationships are bidirectional** - A match from Tier 1 â†’ Tier 3 is equally valid as Tier 3 â†’ Tier 1
2. **Traversal is non-linear** - The LLM may cite T1 â†’ T2 â†’ T3 â†’ T1 â†’ T2 in a single annotation
3. **Skip-tier is valid** - Direct T1 â†” T3 connections are meaningful, not errors
4. **The web is dynamic** - Different taxonomies create different web topologies

---

## Key Design Principles

| Principle | Description |
|-----------|-------------|
| **Per-Chapter Processing** | Workflow executes for EACH chapter in the source book |
| **Source-First** | LLM reviews source chapter BEFORE taxonomy |
| **Metadata-Driven** | Concept matching uses enriched metadata, not raw text |
| **No Arbitrary Limits** | LLM requests ALL relevant chapters |
| **Spider Web Traversal** | Relationships form a bidirectional web, not a one-way hierarchy |
| **Bidirectional References** | ALL tier relationships work in both directions (up/down/across) |
| **Skip-Tier Valid** | Direct connections between non-adjacent tiers are meaningful |
| **Non-Linear Paths** | A single annotation may traverse T1â†’T2â†’T3â†’T1â†’T2 |
| **Tier-Aware Synthesis** | Annotations structured by tier priority (from taxonomy) |
| **Validation Required** | LLM must verify genuine relevance, not just keyword overlap |
| **Gap Acknowledgment** | Missing tier coverage is explicitly noted |
| **Citation Precision** | Chicago-style citations with page numbers |
